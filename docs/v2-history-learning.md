# History & Learning System

## Overview

ShAI includes a SQLite-based history tracking system with learning capabilities. This document explains the implementation, database schema, and user flows.

---

## Features

### 1. Command History Tracking

Every command generated by ShAI (whether from AI or shortcuts) is tracked in a local SQLite database.

**What's Tracked:**
- User's natural language query
- Generated shell command
- Source (AI or shortcut)
- Working directory
- Execution status (executed or not)
- Exit code (if executed)
- Timestamp

### 2. Usage Statistics

Aggregated statistics about command usage:
- Total commands generated
- Execution rate
- AI vs Shortcut breakdown
- Most frequently used commands
- Today/this week counts

### 3. Smart Shortcut Suggestions

The system learns from usage patterns and suggests creating shortcuts for frequently used commands.

**Key Design: Command-Based Tracking**

Stats are keyed by COMMAND, not query. This handles:
- Typos: "find larg files" → same command
- Synonyms: "show big files" → same command
- Different phrasings: all count together

When a command is used 3+ times, it becomes a suggestion candidate.

| Command | Description |

## Database Schema

### Location

```
~/.shai/history.db
```

File permissions: `0o600` (owner read/write only)

### Tables

#### `history` (Auto-cleaned)

Stores recent command history. Automatically cleaned based on retention settings.

```sql
CREATE TABLE IF NOT EXISTS history (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  query TEXT NOT NULL,
  command TEXT NOT NULL,
  source TEXT NOT NULL CHECK(source IN ('ai', 'shortcut')),
  working_directory TEXT NOT NULL,
  executed INTEGER DEFAULT 0,
  exit_code INTEGER,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

| Column | Type | Description |
|--------|------|-------------|
| id | INTEGER | Primary key |
| query | TEXT | User's natural language input |
| command | TEXT | Generated shell command |
| source | TEXT | "ai" or "shortcut" |
| working_directory | TEXT | CWD when command was generated |
| executed | INTEGER | 0=not run, 1=user executed it |
| exit_code | INTEGER | Command exit code (NULL if not executed) |
| created_at | TEXT | ISO timestamp |

**Retention Policy:**
- Default: Delete entries older than 30 days
- Cap at 2000 entries maximum
- Cleanup runs automatically once per day

#### `query_stats` (Never deleted)

Aggregated usage statistics **keyed by COMMAND**.

```sql
CREATE TABLE IF NOT EXISTS query_stats (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  command TEXT UNIQUE NOT NULL,
  source TEXT NOT NULL CHECK(source IN ('ai', 'shortcut')),
  use_count INTEGER DEFAULT 1,
  success_count INTEGER DEFAULT 0,
  suggested INTEGER DEFAULT 0,
  first_used TEXT NOT NULL,
  last_used TEXT NOT NULL
);
```

| Column | Type | Description |
|--------|------|-------------|
| id | INTEGER | Primary key |
| command | TEXT | Unique shell command (UNIQUE KEY) |
| source | TEXT | "ai" or "shortcut" |
| use_count | INTEGER | Total times this command was used |
| success_count | INTEGER | Times it succeeded (exit_code=0) |
| suggested | INTEGER | 0=not suggested, 1=already suggested |
| first_used | TEXT | First time command was used |
| last_used | TEXT | Most recent use |

**Why keyed by command?**
- User types "find large files" → `find . -size +100M` → Entry A
- User types "find larg files" (typo) → same command → Entry A (count++)
- User types "show large files" (synonym) → may generate same command → Entry A (count++)

**Size:** Even a power user has 200-500 unique commands. Table stays under 100KB forever.

#### `metadata`

Key-value store for database settings.

```sql
CREATE TABLE IF NOT EXISTS metadata (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL
);
```

| key | value |
|-----|-------|
| db_version | "1" |
| last_cleanup | "2025-01-08T10:00:00.000Z" |

## Implementation

### Recording Commands (`src/core/history.ts`)

```typescript
export function recordCommand(data: {
  query: string;
  command: string;
  source: 'ai' | 'shortcut';
}): number {
  const db = getDatabase();
  const now = new Date().toISOString();

  // Insert into history table
  const historyResult = db.prepare(
    `INSERT INTO history (query, command, source, working_directory, executed, created_at) 
     VALUES (?, ?, ?, ?, 0, ?)`
  ).run(data.query, data.command, data.source, process.cwd(), now);

  // Update query_stats (keyed by COMMAND)
  const existingStats = db.prepare('SELECT id FROM query_stats WHERE command = ?')
    .get(data.command);

  if (existingStats) {
    db.prepare('UPDATE query_stats SET use_count = use_count + 1, last_used = ? WHERE command = ?')
      .run(now, data.command);
  } else {
    db.prepare(
      `INSERT INTO query_stats (command, source, use_count, success_count, suggested, first_used, last_used)
       VALUES (?, ?, 1, 0, 0, ?, ?)`
    ).run(data.command, data.source, now, now);
  }

  return historyResult.lastInsertRowid as number;
}
```

### Marking Execution (`src/core/history.ts`)

```typescript
export function markExecuted(historyId: number, exitCode: number): void {
  const db = getDatabase();

  // Update history entry
  db.prepare('UPDATE history SET executed = 1, exit_code = ? WHERE id = ?')
    .run(exitCode, historyId);

  // Update success count in query_stats
  const historyRow = db.prepare('SELECT command FROM history WHERE id = ?')
    .get(historyId);

  if (historyRow && exitCode === 0) {
    db.prepare('UPDATE query_stats SET success_count = success_count + 1 WHERE command = ?')
      .run(historyRow.command);
  }
}
```

### Getting Suggestions (`src/core/learning.ts`)

```typescript
export function getShortcutSuggestions(threshold = 3): ShortcutSuggestion[] {
  const db = getDatabase();

  const rows = db.prepare(
    `SELECT command, use_count FROM query_stats 
     WHERE source = 'ai' 
       AND use_count >= ?
       AND suggested = 0
     ORDER BY use_count DESC
     LIMIT 10`
  ).all(threshold);

  return rows.map(row => ({
    command: row.command,
    useCount: row.use_count,
    suggestedName: suggestShortcutNameFromCommand(row.command),
  }));
}

export function suggestShortcutNameFromCommand(command: string): string {
  // Extract meaningful parts from the command
  // "lsof -ti:3000 | xargs kill -9" → "killport"
  const parts = command.split(/[\s\-&|;()]+/).filter(p => p.length > 0);
  const mainCmd = parts[0];
  
  if (parts.length > 1) {
    return `${mainCmd}${parts[1]}`.slice(0, 12);
  }
  return mainCmd.slice(0, 12);
}
```

### Cleanup (`src/core/history.ts`)

```typescript
export function cleanupHistory(config: {
  retentionDays: number;
  maxEntries: number;
}): number {
  const db = getDatabase();

  // Delete old entries
  const deleteByAge = db.prepare(
    `DELETE FROM history WHERE created_at < datetime('now', '-' || ? || ' days')`
  ).run(config.retentionDays);

  // Delete excess entries
  const deleteExcess = db.prepare(
    `DELETE FROM history WHERE id NOT IN (
       SELECT id FROM history ORDER BY created_at DESC LIMIT ?
     )`
  ).run(config.maxEntries);

  setMetadata('last_cleanup', new Date().toISOString());
  return deleteByAge.changes + deleteExcess.changes;
}

export function shouldRunCleanup(): boolean {
  const lastCleanup = getMetadata('last_cleanup');
  if (!lastCleanup || lastCleanup === '0') return true;

  const lastCleanupDate = new Date(lastCleanup);
  const oneDayMs = 24 * 60 * 60 * 1000;
  return Date.now() - lastCleanupDate.getTime() > oneDayMs;
}
```

## Configuration Options

Settings in `~/.shai/config.json`:

```json
{
  "settings": {
    "historyEnabled": true,
    "historyRetentionDays": 30,
    "historyMaxEntries": 2000
  }
}
```

| Setting | Type | Default | Description |
|---------|------|---------|-------------|
| historyEnabled | boolean | true | Enable/disable history tracking |
| historyRetentionDays | number | 30 | Days to keep history entries |
| historyMaxEntries | number | 2000 | Maximum history entries to keep |

## Privacy & Security

### Local-Only Storage
- All data stored locally in `~/.shai/history.db`
- Nothing leaves the machine except AI API calls
- No cloud sync, no telemetry

### File Permissions
- Database file: `0o600` (owner read/write only)
- Same security model as config and shortcuts

### Sensitive Commands
- Users can disable history via config (`historyEnabled: false`)
- Clear history anytime with `shai --clear-history --all`
- Retention policy auto-deletes old entries
